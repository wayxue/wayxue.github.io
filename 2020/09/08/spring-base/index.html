spring

  1. spring 是一个轻量级java 开发框架。spring 为了解决企业级应用开发的业务逻辑和其它各层的耦合问题。spring 是一个IOC 和AOP 的容器框架。IOC ：控制反转。AOP ：面向切面编程。容器：包含并管理应用对象的生命周期。
  2. spring 的有点：spring 通过DI 和AOP 消除样板式代码来简化企业级java 开发。spring 框架之外还存在一个构建在核心框架之上的庞大生态圈，它将spring 扩展到不同的领域，如Web 服务、REST、移动开发以及NoSQL。低侵入式设计，代码的污染极低。独立于各种应用服务器，基于spring 框架的应用，可以真正实现Write Once, Run Anywhere 的承诺。spring 的IOC 容器降低了业务对象替换的复杂性，提高了组件之间的解耦。spring 的AOP 支持允许将一些通用任务如安全、事务、日志等进行集中式处理，从而提供了更好的复用。spring 的ORM 和DAO 提供了与第三方持久层框架的良好整合，并简化了底层数据库的访问。spring 的高度开放性，并不强制应用完全依赖于spring，开发者可自由选用spring 框架的部分或全部。任何一个语言或者任何一个框架想要利于不败之地，那么很重要的就是它的生态。
  3. IOC 是一种设计思想，在java 开发中，将你设计好的对象交给容器控制，而不是显示地用代码进行对象的创建。所谓IOC ：对象有spring 来创建、管理、装配。
  4. 接口分离原则ISP(the Interface Segregation Principle)
  5. 依赖倒置原则DIP(the Dependency Inversion Principle)
  6. IOC 有点：集中管理。功能可复用（减少对象的创建和内存消耗）。使得程序的整个体系机构可维护性、灵活性、扩展性变高。解耦。
